Created the Aetonet repository and pushed

Used gh repo create to create the repository under the Aetonet organization, set it as the origin remote, and push the main branch — all in a single command.

$ gh repo create Aetonet/research-and-findings \ --public \ --description "Aetonet research reports and findings" \ --source . \ --remote origin \ --push https://github.com/Aetonet/research-and-findings ✓ Branch 'main' set up to track 'origin/main'.
09

Enabled GitHub Pages

Activated GitHub Pages via the API, deploying from the root of the main branch. The site went live within ~60 seconds.

$ gh api repos/Aetonet/research-and-findings/pages \ --method POST \ -f 'source[branch]=main' \ -f 'source[path]=/' ✓ html_url: "https://aetonet.github.io/research-and-findings/" ✓ https_enforced: true ✓ source: { branch: "main", path: "/" }

HTTPS enforcement was automatically enabled by GitHub. The site is served over a secure connection at no extra configuration cost.

03 — Key Decisions

Choices made and why

Several decisions were made deliberately during this setup. Recorded here for future reference.

SSH over HTTPS
Chosen at the user's preference for stronger security posture. SSH key authentication eliminates reliance on password or token strings stored in plaintext. The existing id_ed25519_github key was reused — no new key was generated.
Ed25519 key type
The existing key used the Ed25519 algorithm — the current best practice for SSH keys. Faster and more secure than RSA-4096. No action was needed since it was already in place.
Public repository
Repository created as public so GitHub Pages works without a paid plan. GitHub Pages on free/organization accounts requires a public repo for custom deployments.
Flat file structure
All HTML files sit in the repository root alongside index.html. No subdirectories. This keeps GitHub Pages routing simple — every file is directly accessible at /filename.html with no path prefix needed.
Numbered filename prefix
Files use the N_slug.html naming convention. The number keeps files sorted correctly in the filesystem, the OS, and git diffs. The slug provides human-readable context without opening the file.
No build step
Plain HTML files. No static site generator, no bundler, no CI pipeline. GitHub Pages serves them directly from the main branch root. Adding a report is three git commands — no build to trigger or await.
